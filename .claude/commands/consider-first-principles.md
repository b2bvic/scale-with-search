# First Principles Decomposition

## When to Use

Invoke when reasoning feels circular, when conventional wisdom is being accepted without scrutiny, when "that's how it's always been done" is the justification, or when a novel solution requires escaping inherited assumptions.

## Execution

1. **State the problem or belief.** Write it plainly. What are you trying to solve, decide, or understand?

2. **Surface the assumptions.** List every assumption baked into the current approach. These often hide behind phrases like "obviously," "everyone knows," "the standard is," or "we have to." Pull them out and lay them flat.

3. **Challenge each assumption.** For every assumption, ask: Is this a law of physics, or a convention? Is this provably true, or just familiar? Could this be wrong, outdated, or context-dependent?

4. **Identify fundamental truths.** Strip away the challenged assumptions. What remains? These are the irreducible facts — constraints that survive scrutiny. They might be physical laws, mathematical certainties, verified data, or binding legal/contractual realities.

5. **Rebuild from the ground up.** Using only the fundamental truths as building blocks, construct a new approach. Ignore what existed before. Ask: if I were designing this from scratch, knowing only these truths, what would I build?

6. **Compare to the original.** Place the rebuilt solution next to the inherited one. Where do they diverge? The gaps reveal where convention was substituting for reasoning.

## Output Format

```
## First Principles Analysis: [Topic]

### Current Approach
[Brief description of existing solution/belief]

### Assumptions Identified
1. [Assumption] — [Challenged: true constraint / convention / outdated]
2. ...

### Fundamental Truths (what survives)
1. [Truth]
2. ...

### Rebuilt Solution
[New approach constructed from truths alone]

### Divergence from Convention
[Where the rebuilt solution differs and why]
```
